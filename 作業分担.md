# 作業分担（複数Claude Code並行開発）

## 基本方針

依存関係があるため、**Phase 1（基盤）を先に1つのインスタンスで完成させてから**、
Phase 2以降を複数インスタンスで並行作業する。

---

## Phase 1：基盤構築（先行・1インスタンス）

> **担当A** が単独で完了させてから、B・C・Dが並行開始する

### 担当A の作業範囲

| カテゴリ | 作業内容 |
|----------|----------|
| プロジェクト初期化 | `npx create-next-app` + TypeScript設定 |
| Docker | `docker-compose.yml`（PostgreSQL環境） |
| Prisma | `prisma/schema.prisma`（全テーブル定義） + 初期マイグレーション |
| 認証 | `src/lib/auth.ts`（NextAuth + 氏名+生年月日認証） |
| 共通UI | `src/app/(main)/layout.tsx`（サイドバー・ヘッダー） |
| ログイン | `src/app/(auth)/login/page.tsx` |
| shadcn/ui | 基本コンポーネントのインストール・設定 |
| Prismaクライアント | `src/lib/prisma.ts` |
| 型定義 | `src/types/index.ts`（共通型） |
| 環境変数テンプレート | `.env.local.example` |

### 担当Aの完了条件

- [ ] `docker-compose up` でPostgreSQLが起動する
- [ ] `npx prisma migrate dev` でマイグレーションが通る
- [ ] ログイン画面が表示され、認証が機能する
- [ ] サイドバー付きの共通レイアウトが表示される

---

## Phase 2：機能開発（並行作業）

担当Aの完了後、B・C・Dが**同時に**作業開始する。

## 作業の流れ

```
[担当A] 基盤構築
   ↓ 完了後
   ├──[担当B] 生徒管理・固定引継事項
   ├──[担当C] 引継書・ファイル添付        ← 同時並行
   └──[担当D] AIレポート・管理者・Dashboard
   ↓ 全員完了後
[統合テスト・デプロイ準備]
```

---

## 共有ルール

1. **Prismaスキーマは変更しない**（担当Aが定義したものを使う）
2. **`src/lib/auth.ts` は変更しない**（必要なら担当Aに依頼）
3. **`src/app/(main)/layout.tsx` は変更しない**
4. **`src/components/ui/`はshadcn/uiのコンポーネントをそのまま使う**（新規追加のみ可）
5. 他の担当のファイルを読むのはOK・変更はNG
