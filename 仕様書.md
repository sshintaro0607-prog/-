# 授業引継書システム 仕様書

**バージョン**: 1.0  
**作成日**: 2026-04-03  
**対象**: 学習塾向け 授業引継書デジタル管理システム

---

## 1. システム概要

### 1.1 目的

本システムは、学習塾における講師間の授業引継情報を一元管理し、講師交代時の情報伝達を円滑化するためのWebアプリケーションである。従来の紙・口頭ベースの引継を廃し、リアルタイムでの情報共有・蓄積・AI活用による月次レポート自動生成を実現する。

### 1.2 背景・課題

- 講師交代時に口頭や紙で引継を行っており、情報の欠落・誤伝達が発生している
- 生徒ごとの状況・注意事項が属人化しており、新任講師が把握するまでに時間がかかる
- 過去の授業記録を参照する手段がなく、継続的な指導方針の維持が困難
- 室長が全講師・全生徒の状況を俯瞰的に把握する仕組みが存在しない

### 1.3 システム範囲

| 対象 | 内容 |
|------|------|
| 対象ユーザー | 室長（1名以上）、講師（複数名） |
| 対象データ | 生徒情報（基本のみ）、授業引継書、固定引継事項、添付ファイル、AIレポート |
| 対象外 | 保護者向けポータル、請求・料金管理、出席管理、生徒の詳細情報（他の基幹システムで管理） |

## 3. 機能要件

### 3.1 認証・アカウント管理

#### 3.1.1 ログイン

- **講師**: 講師氏名 + 生年月日（YYYY-MM-DD）でログイン
- **室長**: 管理者専用のID + パスワードでログイン
- アカウントロック機能なし
- セッション有効期限：8時間

#### 3.1.2 アカウント管理（室長のみ）

- 講師アカウントの新規作成（名前、生年月日）
- 講師アカウントの無効化・有効化（論理削除）
- 講師一覧の表示

### 3.2 生徒管理

#### 3.2.1 生徒登録・編集（室長のみ）

登録項目：
- 生徒番号（必須、一意）
- 氏名（必須）
- 氏名かな（必須）
- 学年（必須）：小1〜小6、中1〜中3、高1〜高3から選択

> ※ その他の生徒情報（連絡先、保護者情報等）は別の基幹システムで管理する

#### 3.2.2 生徒一覧

- 全講師が参照可能
- 検索：生徒番号、生徒名（かな含む）
- フィルタリング：学年
- ソート：氏名かな順（デフォルト）、生徒番号順

#### 3.2.3 生徒詳細

- 生徒基本情報の表示
- 固定引継事項の表示・編集
- その生徒の引継書一覧（新しい順）
- その生徒に関連するAIレポートへのリンク

### 3.3 授業引継書（メイン機能）

#### 3.3.1 引継書作成

**必須項目**：
- 対象生徒（生徒選択）
- 授業日（日付ピッカー、デフォルト：本日）
- 科目（国語・数学・英語・理科・社会・その他）

**授業記録項目**（構造化フィールド）：
- 今日の内容
- できたこと
- できなかったこと
- できなかったことに対する改善策
- 特記事項

**ファイル添付**（任意、複数可）

**固定引継事項の表示**：生徒に設定されている固定引継事項を参照用として常に表示

作成後の動作：
- 保存と同時に他のユーザーが参照可能（承認フローなし）
- 作成者・作成日時を自動記録

#### 3.3.2 引継書一覧

- 全ユーザーが参照可能
- 検索：生徒名、キーワード（授業内容等）
- フィルタリング：生徒、科目、作成者、授業日（期間指定）
- ソート：授業日降順（デフォルト）
- ページネーション：20件/ページ

#### 3.3.3 引継書詳細

- 全項目の表示
- 固定引継事項を引継書の上部に常時表示
- 添付ファイルの一覧とダウンロード
- 編集ボタン：室長は全件、講師は自作成分のみ
- 削除ボタン：室長のみ（論理削除）

#### 3.3.4 引継書編集

- 作成者（講師）または室長のみ編集可能
- 編集履歴（最終更新者・更新日時）を記録・表示
- 添付ファイルの追加・削除

### 3.4 固定引継事項

- 生徒ごとに「毎回引き継ぎたいこと」を登録・管理
- 引継書の作成画面・詳細画面で常に表示される常設メモ欄
- 室長・全講師が編集可能（最終更新者・更新日時を記録）
- 複数の固定事項を登録可能（例：アレルギー情報、苦手単元、保護者への注意事項等）

### 3.5 ファイル・資料添付

- 対応フォーマット：PDF、Word（.docx）、Excel（.xlsx）、PowerPoint（.pptx）、画像（PNG、JPEG、GIF、WebP）
- 最大ファイルサイズ：1ファイルあたり20MB
- 1引継書あたりの添付上限：10ファイル
- 認証済みユーザーのみダウンロード可能（直接URLアクセス不可）
- ストレージ：サーバーローカルストレージ（本番環境ではAWS S3推奨）

### 3.6 AI月次まとめレポート

#### 3.6.1 概要

- Anthropic Claude API（claude-sonnet-4-5）を使用
- 対象期間：前月1日〜末日の引継書データを集計・分析
- 生成単位：生徒ごと、または塾全体

#### 3.6.2 生成タイミング

- 毎月1日 AM 9:00 に全生徒分を自動生成（cron job）
- 室長が管理画面から任意のタイミングで手動生成可能

#### 3.6.3 AIレポートの内容

```
1. 学習進捗サマリー
   - 当月扱った単元・範囲の概要
   - 科目別の進捗状況

2. できたこと・できなかったことの傾向
   - 繰り返し「できたこと」のパターン（定着している単元）
   - 繰り返し「できなかったこと」のパターン（課題）

3. 改善策の集約
   - 引継書で記録された改善策の共通点・優先事項

4. 来月の指導方針案
   - 引継書の内容を踏まえた次月の推奨アクション
   - 優先して取り組むべき単元・課題

5. 授業頻度（引継書の件数から推計）
```

## 4. データモデル

### 4.1 テーブル定義

#### `users`（ユーザー）

```sql
CREATE TABLE users (
  id            UUID          PRIMARY KEY DEFAULT gen_random_uuid(),
  name          VARCHAR(100)  NOT NULL,
  birth_date    DATE,                          -- 講師ログイン用（室長はNULL可）
  role          VARCHAR(20)   NOT NULL CHECK (role IN ('ADMIN', 'TEACHER')),
  password_hash VARCHAR(255),                  -- 室長のみ使用
  is_active     BOOLEAN       NOT NULL DEFAULT true,
  created_at    TIMESTAMPTZ   NOT NULL DEFAULT NOW(),
  updated_at    TIMESTAMPTZ   NOT NULL DEFAULT NOW()
);
```

#### `students`（生徒）

```sql
CREATE TABLE students (
  id             UUID          PRIMARY KEY DEFAULT gen_random_uuid(),
  student_number VARCHAR(20)   NOT NULL UNIQUE,  -- 生徒番号
  name           VARCHAR(100)  NOT NULL,
  name_kana      VARCHAR(100)  NOT NULL,
  grade          VARCHAR(20)   NOT NULL,
  -- 例: 小1〜小6, 中1〜中3, 高1〜高3
  is_active      BOOLEAN       NOT NULL DEFAULT true,
  created_at     TIMESTAMPTZ   NOT NULL DEFAULT NOW(),
  updated_at     TIMESTAMPTZ   NOT NULL DEFAULT NOW()
);
```

#### `student_standing_notes`（固定引継事項）

```sql
CREATE TABLE student_standing_notes (
  id           UUID        PRIMARY KEY DEFAULT gen_random_uuid(),
  student_id   UUID        NOT NULL REFERENCES students(id) ON DELETE CASCADE,
  content      TEXT        NOT NULL,
  sort_order   INTEGER     NOT NULL DEFAULT 0,
  updated_by   UUID        NOT NULL REFERENCES users(id),
  created_at   TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at   TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

#### `handover_records`（授業引継書）

```sql
CREATE TABLE handover_records (
  id              UUID        PRIMARY KEY DEFAULT gen_random_uuid(),
  student_id      UUID        NOT NULL REFERENCES students(id),
  author_id       UUID        NOT NULL REFERENCES users(id),
  subject         VARCHAR(20) NOT NULL
                  CHECK (subject IN ('JAPANESE','MATH','ENGLISH','SCIENCE','SOCIAL','OTHER')),
  lesson_date     DATE        NOT NULL,
  todays_content  TEXT,       -- 今日の内容
  achieved        TEXT,       -- できたこと
  not_achieved    TEXT,       -- できなかったこと
  improvement     TEXT,       -- 改善策
  special_notes   TEXT,       -- 特記事項
  last_updated_by UUID        REFERENCES users(id),
  is_deleted      BOOLEAN     NOT NULL DEFAULT false,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_handover_student_id  ON handover_records(student_id);
CREATE INDEX idx_handover_lesson_date ON handover_records(lesson_date DESC);
CREATE INDEX idx_handover_subject     ON handover_records(subject);
```

#### `attachments`（添付ファイル）

```sql
CREATE TABLE attachments (
  id            UUID         PRIMARY KEY DEFAULT gen_random_uuid(),
  handover_id   UUID         NOT NULL REFERENCES handover_records(id) ON DELETE CASCADE,
  uploaded_by   UUID         NOT NULL REFERENCES users(id),
  original_name VARCHAR(255) NOT NULL,
  stored_name   VARCHAR(255) NOT NULL UNIQUE,
  storage_path  TEXT         NOT NULL,
  mime_type     VARCHAR(100) NOT NULL,
  file_size     BIGINT       NOT NULL,
  is_deleted    BOOLEAN      NOT NULL DEFAULT false,
  created_at    TIMESTAMPTZ  NOT NULL DEFAULT NOW()
);
```

#### `ai_reports`（AIレポート）

```sql
CREATE TABLE ai_reports (
  id                UUID        PRIMARY KEY DEFAULT gen_random_uuid(),
  student_id        UUID        REFERENCES students(id),  -- NULLは塾全体レポート
  report_type       VARCHAR(20) NOT NULL CHECK (report_type IN ('STUDENT', 'OVERALL')),
  target_year       SMALLINT    NOT NULL,
  target_month      SMALLINT    NOT NULL CHECK (target_month BETWEEN 1 AND 12),
  content           TEXT        NOT NULL,    -- Markdown形式
  generated_by      UUID        REFERENCES users(id),  -- NULLは自動生成
  model_used        VARCHAR(100) NOT NULL,
  created_at        TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  UNIQUE(student_id, target_year, target_month)
);
```

#### `system_settings`（システム設定）

```sql
CREATE TABLE system_settings (
  key        VARCHAR(100) PRIMARY KEY,
  value      TEXT         NOT NULL,
  updated_by UUID         REFERENCES users(id),
  updated_at TIMESTAMPTZ  NOT NULL DEFAULT NOW()
);
```

## 7. 主要APIエンドポイント

### 生徒API

| メソッド | パス | 説明 | 権限 |
|----------|------|------|------|
| GET | `/api/students` | 生徒一覧（クエリ：search, grade, page） | 認証済み |
| POST | `/api/students` | 生徒登録 | ADMIN |
| GET | `/api/students/[id]` | 生徒詳細 | 認証済み |
| PATCH | `/api/students/[id]` | 生徒情報更新 | ADMIN |
| DELETE | `/api/students/[id]` | 生徒の論理削除 | ADMIN |

| PUT | `/api/students/[id]/standing-notes` | 固定引継事項更新 | 認証済み |

## 10. 用語集
